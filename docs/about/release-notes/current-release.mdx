NeMo Platform v0.2.0 is the OSS 0.2 release line for local quickstart,
Docker-backed testing, SDK validation, and plugin-based platform workflows.
This release expands the first public release with fine-tuning, local model
deployment, synthetic data generation, model auditing, and access-control work.

## Release status

- Development complete: June 12, 2026
- QA start: June 15, 2026
- Target release date: June 25, 2026
- Version markers: platform API `0.2.0`, SDK `0.2.0`

## Highlights

- **Fine-tuning.** Customizer is now part of the plugin-based platform
  architecture, with Docker-backed job execution, Unsloth and NeMo Automodel
  support, vLLM adapter support, and updated documentation.
- **Docker-backed local runtime.** The platform 0.2 scope includes Docker
  execution paths for the platform, customization jobs, and local model
  deployment testing.
- **Model deployment.** vLLM and NIM deployment flows are covered by the 0.2
  model-deployment workstream, including vLLM deployment in Docker.
- **Synthetic data generation.** Safe Synthesizer moves into the platform as a
  local plugin for privacy-preserving synthetic data generation.
- **Model auditing.** The local garak-backed Auditor plugin has documentation
  and validation work for QA.
- **Access control.** Auth and authorization work continues with plugin
  authorization support and end-to-end auth wiring in QA.
- **Platform quality.** The 0.2 line includes fixes for seven tracked
  platform bugs.

## What's included

### Platform and Plugins

- Plugin support for platform capabilities that ship outside the core
  package.
- Authorization support for plugins.
- Alignment work between human documentation and coding-agent skill
  resources.
- Docker support for running the platform locally.

### Fine-tune Models

- Customizer migration to the plugin architecture.
- Docker-backed Customizer jobs.
- Unsloth and NeMo Automodel support.
- vLLM adapter support and end-to-end vLLM testing.
- Documentation updates for the 0.2 Customizer flow.

### Models and Inference

- vLLM deployment in Docker.
- NIM deployment in Docker.
- Local model-deployment validation for the OSS quickstart and SDK testing
  paths.

### Agents

- `nemo agents` container subcommands for rendering, building, and publishing
  agent containers.
- Continued NAT-based agent workflow support from the 0.1 line.

### Safe Synthesizer

- Safe Synthesizer is packaged as a platform plugin.
- Local execution support for Safe Synthesizer workflows.
- Documentation and tutorials for data synthesis, PII replacement,
  evaluation, jobs, host-local development, and differential privacy.

### Auditor

- Auditor plugin documentation for local garak-backed auditing.
- Auditor plugin skill fixes.
- QA handoff for automated tests and validation.

### Evaluation

- Evaluator SDK and plugin enhancement work is tracked in the 0.2 line.

## Install

For a fresh local checkout:

```bash
git clone https://github.com/NVIDIA-NeMo/nemo-platform.git
cd nemo-platform
git checkout v0.2.0
make bootstrap
nemo setup
```

See [Setup](/documentation/get-started) for prerequisites and provider
configuration.

## Upgrade from v0.1.x

From an existing local checkout:

```bash
git fetch --tags
git checkout v0.2.0
make bootstrap
nemo setup
```

After setup, restart local services before running CLI, SDK, Studio, or plugin
smoke tests against the upgraded checkout.

## Compatibility

- Python 3.11-3.14
- macOS and Linux for the OSS local install path
- Docker for Docker-backed platform, job, and model-deployment workflows
- NVIDIA GPU access for local training, model deployment, and GPU-backed
  synthetic data workflows
- Node 22.18.0+ for Studio assets
- Platform API `0.2.0` and `nemo-platform` Python SDK `0.2.0`

## Known constraints

- **Release date.** The 0.2 release line is targeted for June 25, 2026. If you
  are testing before that date, use the release branch or candidate build
  provided by the release owner.
- **Customizer ARM64 and automodel Docker support.** Docker-backed Customizer
  jobs are validated, but ARM64 build support and automodel Docker support are
  still pending in QA notes.
- **vLLM Docker validation.** vLLM adapter support and Docker deployment are in
  the 0.2 scope. Validate the vLLM image, GPU runtime, and host driver stack in
  your target environment.
- **Safe Synthesizer local mode.** Safe Synthesizer is part of the 0.2 plugin
  workstream and was in QA as of June 9, 2026.
- **Auditor local mode.** Auditor documentation was handed to QA for automated
  test and validation coverage.
- **Auth and RBAC.** Auth and plugin authorization work is in QA and should be
  validated for your deployment mode before relying on it for multi-user
  access control.
- **Evaluator Skill Evaluation MVP.** Skill Evaluation MVP was descoped from
  v0.2.0.
- **Local-first OSS scope.** v0.2.0 improves Docker-backed local workflows. It
  is not a managed cluster or scale-out enterprise release.
